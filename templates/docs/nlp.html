Text Classification

Our text classification services allow you to categorize text into predefined categories. This is useful for content categorization, topic identification, intent recognition, and more.

Available Endpoints

POST /nlp/classify Simple classification of text into one of the provided categories
POST /nlp/classify/multi Multi-class classification with confidence scores for each category

Simple Classification

The Simple Classification endpoint categorizes text into a single class from a provided list of categories.

Request Parameters
  • categories (required) - Array of categories to classify into
  • user_input (required) - Text to classify
  • model (optional) - LLM model to use (gpt-4o-mini or gpt-4o, default: gpt-4o-mini)
Example Request
POST /nlp/classify
Headers:
  X-Token: your_token_here
  Content-Type: application/json

{
  "categories": ["sports", "politics", "technology", "entertainment"],
  "user_input": "Apple announced its latest iPhone with improved AI capabilities at yesterday's event in Cupertino. The new device features enhanced processing power and camera quality.",
  "model": "gpt-4o-mini"
}
Example Response
{
  "result_class": "technology",
  "model_used": "gpt-4o-mini",
  "input_tokens": 125,
  "completion_tokens": 42,
  "output_tokens": 167
}

Multi-Class Classification

The Multi-Class Classification endpoint provides confidence scores for all provided categories, allowing for more nuanced classification.

Request Parameters
  • categories (required) - Array of categories to classify into
  • user_input (required) - Text to classify
  • model (optional) - LLM model to use (gpt-4o-mini or gpt-4o, default: gpt-4o-mini)
Example Request
POST /nlp/classify/multi
Headers:
  X-Token: your_token_here
  Content-Type: application/json

{
  "categories": ["sports", "politics", "technology", "entertainment"],
  "user_input": "Apple announced its latest iPhone with improved AI capabilities at yesterday's event in Cupertino. The new device features enhanced processing power and camera quality.",
  "model": "gpt-4o-mini"
}
Example Response
{
  "top_result": "technology",
  "top_3_classes": ["technology", "entertainment", "politics"],
  "all_classes": [
    {"category": "technology", "confidence": 0.95},
    {"category": "entertainment", "confidence": 0.45},
    {"category": "politics", "confidence": 0.15},
    {"category": "sports", "confidence": 0.05}
  ],
  "model_used": "gpt-4o-mini",
  "input_tokens": 145,
  "completion_tokens": 87,
  "output_tokens": 232
}

Pro Tip: Text Classification

For optimal classification results:

  • Use distinct, non-overlapping categories when possible
  • Provide sufficient context in your input text
  • For complex classification tasks, use the gpt-4o model
  • For multiclass classification, confidence scores don't need to sum to 1.0 - they represent each category's relevance independently
  • The top_3_classes array provides a quick way to identify the most relevant categories

Sentiment Analysis

Our sentiment analysis services analyze text to determine the emotional tone and attitude expressed. This is useful for brand monitoring, customer feedback analysis, social media monitoring, and more.

Available Endpoints

POST /nlp/sentiment Basic sentiment analysis (positive, neutral, negative)
POST /nlp/sentiment/advanced Advanced emotion analysis with multiple emotion categories

Basic Sentiment Analysis

The Basic Sentiment Analysis endpoint categorizes text into positive, neutral, or negative sentiment with confidence scores.

Request Parameters
  • user_input (required) - Text to analyze for sentiment
  • model (optional) - LLM model to use (gpt-4o-mini or gpt-4o, default: gpt-4o-mini)
Example Request
POST /nlp/sentiment
Headers:
  X-Token: your_token_here
  Content-Type: application/json

{
  "user_input": "I absolutely love this product! It's the best purchase I've made this year and has significantly improved my daily workflow.",
  "model": "gpt-4o-mini"
}
Example Response
{
  "top_sentiment": "positive",
  "all_sentiments": [
    {"sentiment": "positive", "confidence": 0.95},
    {"sentiment": "neutral", "confidence": 0.05},
    {"sentiment": "negative", "confidence": 0.00}
  ],
  "model_used": "gpt-4o-mini",
  "input_tokens": 98,
  "completion_tokens": 68,
  "output_tokens": 166
}

Advanced Emotion Analysis

The Advanced Emotion Analysis endpoint provides more granular analysis of emotions present in text, detecting multiple emotional states with confidence scores.

Request Parameters
  • user_input (required) - Text to analyze for emotions
  • model (optional) - LLM model to use (gpt-4o-mini or gpt-4o, default: gpt-4o-mini)
Example Request
POST /nlp/sentiment/advanced
Headers:
  X-Token: your_token_here
  Content-Type: application/json

{
  "user_input": "I was initially anxious about the presentation, but after receiving such positive feedback from my colleagues, I'm feeling incredibly proud and relieved!",
  "model": "gpt-4o-mini"
}
Example Response
{
  "top_emotion": "pride",
  "top_3_emotions": ["pride", "relief", "happiness"],
  "all_emotions": [
    {"emotion": "pride", "confidence": 0.85},
    {"emotion": "relief", "confidence": 0.80},
    {"emotion": "happiness", "confidence": 0.75},
    {"emotion": "anxiety", "confidence": 0.40},
    {"emotion": "satisfaction", "confidence": 0.35}
  ],
  "model_used": "gpt-4o-mini",
  "input_tokens": 115,
  "completion_tokens": 95,
  "output_tokens": 210
}
Supported Emotions

The advanced emotion analysis can detect a wide range of emotions, including:

Primary Emotions

happiness, sadness, anger, fear, surprise, disgust

Positive Emotions

joy, excitement, contentment, satisfaction, relief, pride, optimism, love, affection, compassion, gratitude, admiration

Negative Emotions

grief, disappointment, regret, shame, guilt, remorse, frustration, irritation, rage, contempt, jealousy, envy, anxiety, worry, nervousness, dread, horror, panic

Complex Emotions

hope, anticipation, curiosity, interest, awe, wonder, trust, confidence, doubt, uncertainty, skepticism, nostalgia, longing, yearning, desire, craving

Pro Tip: Sentiment Analysis

For optimal sentiment analysis results:

  • Provide sufficient context in your input text
  • For nuanced emotion analysis, use the advanced endpoint
  • If you need to detect ambivalent or mixed emotions, use the advanced endpoint and look for multiple emotions with high confidence scores
  • The basic sentiment endpoint is faster and more efficient for simple positive/negative/neutral classification
  • For analyzing customer feedback or reviews, consider using gpt-4o for better accuracy in detecting subtle emotional cues

Credit Usage

NLP services consume AI credits based on the model used:

  • Using gpt-4o-mini: 0.5 credits per call
  • Using gpt-4o: 2 credits per call
