File Management Services

Upload, access, and delete files used with GAIA's various AI services. GAIA's file management system provides secure and reliable handling of your data files with Azure Blob Storage.

Download Postman Collection Swagger API Docs

Overview

GAIA's File Management endpoints allow you to upload, access, and delete files used with the variety of available AI services. These utilities are essential for processing audio files for transcription, documents for intelligence analysis, and other data that requires AI processing.


File Storage System

Our file management system uses Azure Blob Storage for secure and reliable file handling:

  • Files are stored temporarily while being processed by our AI services
  • User access is restricted to only their own files (except for administrators)
  • Files used with certain services (like speech-to-text) are automatically deleted after processing
  • All files are securely transmitted and stored using encryption

Secure Upload

Upload files securely with automatic validation and filtering of potentially harmful file types. Each file receives a unique ID for reference in other API calls.

Easy Access

Retrieve secure access URLs to your uploaded files when needed. User permissions ensure only authorized users can access each file.

Simple Management

Delete files when they're no longer needed or let the system automatically clean up files after they've been processed by AI services.

File Listing

View all your uploaded files in one place with detailed information including file names, sizes, content types, and upload dates.

Available Endpoints

POST /file Upload one or more files to the platform
GET /file/url Get access URL for a previously uploaded file
DELETE /file Delete a previously uploaded file
GET /file/list List all files uploaded by the user

Uploading Files

To upload files to GAIA, use the /file endpoint with a multipart form data request containing the file(s) you want to upload.

# Upload a file using cURL curl -X POST https://api.example.com/file \ -H "X-Token: your_token_here" \ -F "files=@document.pdf" \ -F "files=@image.jpg" # Example response { "message": "Files uploaded successfully", "uploaded_files": [ { "file_name": "document.pdf", "file_id": "12345678-1234-5678-1234-567812345678", "content_type": "application/pdf" }, { "file_name": "image.jpg", "file_id": "87654321-8765-4321-8765-432187654321", "content_type": "image/jpeg" } ] }
# Upload a file using Python requests import requests url = "https://api.example.com/file" headers = { "X-Token": "your_token_here" } files = [ ("files", ("document.pdf", open("document.pdf", "rb"), "application/pdf")), ("files", ("image.jpg", open("image.jpg", "rb"), "image/jpeg")) ] response = requests.post(url, headers=headers, files=files) print(response.json()) # Example response # { # "message": "Files uploaded successfully", # "uploaded_files": [ # { # "file_name": "document.pdf", # "file_id": "12345678-1234-5678-1234-567812345678", # "content_type": "application/pdf" # }, # ... # ] # }
// Upload a file using JavaScript fetch const uploadFiles = async () => { const formData = new FormData(); // Add files to form data (from file input) const fileInput = document.getElementById('fileInput'); for (const file of fileInput.files) { formData.append('files', file); } try { const response = await fetch('https://api.example.com/file', { method: 'POST', headers: { 'X-Token': 'your_token_here' }, body: formData }); const result = await response.json(); console.log(result); // Process the file IDs from the result } catch (error) { console.error('Error uploading files:', error); } }; // Example response: // { // "message": "Files uploaded successfully", // "uploaded_files": [ // { // "file_name": "document.pdf", // "file_id": "12345678-1234-5678-1234-567812345678", // "content_type": "application/pdf" // }, // ... // ] // }

The file_id returned in the response is used to reference the file in other API calls.

Accessing Files

To get the URL for a previously uploaded file, use the /file/url endpoint with the file ID you received when uploading.

cURL Example
Python Example
JavaScript Example
# Get file URL using cURL curl -X GET "https://api.example.com/file/url?file_id=12345678-1234-5678-1234-567812345678" \ -H "X-Token: your_token_here" # Example response { "file_name": "document.pdf", "file_url": "https://storage-account.blob.core.windows.net/file-uploads/12345678-1234-5678-1234-567812345678.pdf", "content_type": "application/pdf", "upload_date": "2025-03-12T14:30:45.123Z" }
# Get file URL using Python requests import requests url = "https://api.example.com/file/url" params = { "file_id": "12345678-1234-5678-1234-567812345678" } headers = { "X-Token": "your_token_here" } response = requests.get(url, params=params, headers=headers) file_info = response.json() print(file_info) # Example usage of the file URL file_url = file_info["file_url"] file_content = requests.get(file_url).content # Process file_content as needed
// Get file URL using JavaScript fetch const getFileUrl = async (fileId) => { try { const response = await fetch(`https://api.example.com/file/url?file_id=${fileId}`, { method: 'GET', headers: { 'X-Token': 'your_token_here' } }); const fileInfo = await response.json(); console.log(fileInfo); // Use the file URL const fileUrl = fileInfo.file_url; // Example: Display an image if (fileInfo.content_type.startsWith('image/')) { const imgElement = document.createElement('img'); imgElement.src = fileUrl; document.body.appendChild(imgElement); } // Or download the file const downloadLink = document.createElement('a'); downloadLink.href = fileUrl; downloadLink.download = fileInfo.file_name; downloadLink.click(); } catch (error) { console.error('Error getting file URL:', error); } };

Deleting Files

To delete a file when it's no longer needed, use the /file endpoint with the DELETE method and the file ID.

cURL Example
Python Example
JavaScript Example
# Delete a file using cURL curl -X DELETE "https://api.example.com/file?file_id=12345678-1234-5678-1234-567812345678" \ -H "X-Token: your_token_here" # Example response { "message": "File deleted successfully", "file_id": "12345678-1234-5678-1234-567812345678" }
# Delete a file using Python requests import requests url = "https://api.example.com/file" params = { "file_id": "12345678-1234-5678-1234-567812345678" } headers = { "X-Token": "your_token_here" } response = requests.delete(url, params=params, headers=headers) result = response.json() print(result) # Check if deletion was successful if response.status_code == 200: print(f"File {result['file_id']} deleted successfully") else: print(f"Error deleting file: {result.get('message')}")
// Delete a file using JavaScript fetch const deleteFile = async (fileId) => { try { const response = await fetch(`https://api.example.com/file?file_id=${fileId}`, { method: 'DELETE', headers: { 'X-Token': 'your_token_here' } }); const result = await response.json(); console.log(result); if (response.ok) { // Handle successful deletion console.log(`File ${result.file_id} deleted successfully`); } else { // Handle error console.error(`Error deleting file: ${result.message}`); } } catch (error) { console.error('Error deleting file:', error); } };

Listing Files

To retrieve a list of all files you've uploaded, use the /file/list endpoint. Admin users can see all files across the system.

# List files using cURL with token curl -X GET "https://api.example.com/file/list" \ -H "X-Token: your_token_here" # Or with API key curl -X GET "https://api.example.com/file/list" \ -H "API-Key: your_api_key_here" # Example response { "files": [ { "file_id": "12345678-1234-5678-1234-567812345678", "file_name": "document.pdf", "content_type": "application/pdf", "upload_date": "2025-03-12T14:30:45.123Z", "file_size": 1048576, "blob_url": "https://storage-account.blob.core.windows.net/file-uploads/12345678-1234-5678-1234-567812345678.pdf" }, { "file_id": "87654321-8765-4321-8765-432187654321", "file_name": "image.jpg", "content_type": "image/jpeg", "upload_date": "2025-03-12T15:45:12.987Z", "file_size": 524288, "blob_url": "https://storage-account.blob.core.windows.net/file-uploads/87654321-8765-4321-8765-432187654321.jpg" } ] }

For admin users, the response also includes user_id and user_name fields to identify which user uploaded each file.

Images

Audio

MP3, WAV, M4A, FLAC, OGG

Other

ZIP, XML, JSON, CSV

File Size Limits

Please note the following file size limitations:

  • Maximum file size: 200MB per file
  • For speech-to-text services: Maximum audio length of 2 hours
  • For document intelligence: Maximum of 100 pages per document

For larger files, consider splitting them into smaller chunks or contact our support team for assistance.

Forbidden File Types

For security reasons, the following file types are not allowed:

  • Executable files (.exe, .bat, .cmd, .msi, etc.)
  • Script files (.sh, .ps1, .vbs, .js, etc.)
  • System files (.dll, .sys, etc.)
  • Archive files (.zip, .rar, .7z, etc.)
  • Any other potentially harmful file types
