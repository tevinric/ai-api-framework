File Management

The File Management endpoints allow you to upload, access, and delete files used with our various AI services. These utilities are essential for processing audio files for transcription, documents for intelligence analysis, and other data that requires AI processing.

File Storage System

Our file management system uses Azure Blob Storage for secure and reliable file handling:

  • Files are stored temporarily while being processed by our AI services
  • User access is restricted to only their own files (except for administrators)
  • Files used with certain services (like speech-to-text) are automatically deleted after processing
  • All files are securely transmitted and stored using encryption

Available Endpoints

POST /file Upload one or more files to the platform
GET /file/url Get access URL for a previously uploaded file
DELETE /file Delete a previously uploaded file

Uploading Files

To upload files to our platform, use the /file endpoint with a multipart form data request containing the file(s) you want to upload.

# Example request to upload a file POST /file Headers: X-Token: your_token_here Content-Type: multipart/form-data Form data: files: [file1.pdf, file2.jpg, etc.] # Example response { "message": "Files uploaded successfully", "uploaded_files": [ { "file_name": "document.pdf", "file_id": "12345678-1234-5678-1234-567812345678", "content_type": "application/pdf" }, { "file_name": "image.jpg", "file_id": "87654321-8765-4321-8765-432187654321", "content_type": "image/jpeg" } ] }

The file_id returned in the response is used to reference the file in other API calls.

Accessing Files

To get the URL for a previously uploaded file, use the /file/url endpoint:

# Example request to get file URL GET /file/url Headers: X-Token: your_token_here Content-Type: application/json Body: { "file_id": "12345678-1234-5678-1234-567812345678" } # Example response { "file_name": "document.pdf", "file_url": "https://storage-account.blob.core.windows.net/file-uploads/12345678-1234-5678-1234-567812345678.pdf", "content_type": "application/pdf", "upload_date": "2025-03-12T14:30:45.123Z" }

Deleting Files

To delete a file when it's no longer needed, use the /file endpoint with the DELETE method:

# Example request to delete a file DELETE /file Headers: X-Token: your_token_here Content-Type: application/json Body: { "file_id": "12345678-1234-5678-1234-567812345678" } # Example response { "message": "File deleted successfully", "file_id": "12345678-1234-5678-1234-567812345678" }

Supported File Types

Our platform supports a wide range of file types, including but not limited to:

  • Documents: PDF, DOCX, TXT, RTF, CSV, JSON
  • Images: JPEG, PNG, GIF, BMP, SVG
  • Audio: MP3, WAV, M4A, FLAC, OGG
  • Other: ZIP, XML, etc.

File Size Limits

Please note the following file size limitations:

  • Maximum file size: 100MB per file
  • For speech-to-text services: Maximum audio length of 2 hours
  • For document intelligence: Maximum of 500 pages per document

For larger files, consider splitting them into smaller chunks or contact our support team for assistance.

File Security

We take the security of your files seriously:

  • All files are encrypted both in transit and at rest
  • Access controls ensure users can only access their own files
  • Automatic file deletion after processing helps minimize data exposure
  • Regular security audits verify the integrity of our file storage system
