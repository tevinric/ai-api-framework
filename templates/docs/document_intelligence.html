Document Intelligence

The Document Intelligence API provides powerful tools for extracting information, analyzing layout, and summarizing documents. These services help you transform unstructured document data into structured information that can be easily processed and analyzed.

Download Postman Collection

Authentication Required

All Document Intelligence endpoints require authentication using a token. You must include your token in the X-Token header for all requests.

See the Token Services documentation for details on token generation.

Layout Analysis

The Layout Analysis endpoint extracts structured information from documents, including tables, checkboxes, and text layout. This service is useful for forms, invoices, and documents with complex formatting.

Endpoint

POST /docint/layout Extract tables, checkboxes, and text from forms and documents

Request Parameters

  • file_ids (required) - Array of file IDs to process (uploaded via /file endpoint)
  • include_barcode (optional) - Enable barcode detection (default: false)
  • include_language (optional) - Include language identification (default: false)
  • include_keyvalue_pairs (optional) - Include key-value pairs extraction (default: false)
  • output_format (optional) - Output format for tables and structured content (text or markdown, default: text)
  • page_selection (optional) - Page selection strategy (all_pages or range, default: all_pages)
  • page_range (optional) - Page range to process (e.g., "1,3-6"), only used if page_selection is "range"

Supported File Types

  • PDF (.pdf)
  • Images (.jpg, .jpeg, .png, .tiff, .tif)

Example Request

POST /docint/layout
Headers:
  X-Token: your_token_here
  Content-Type: application/json

{
  "file_ids": ["81f2d4b2-f79c-4d33-b319-e25e658a0fcd"],
  "include_barcode": true,
  "include_keyvalue_pairs": true,
  "output_format": "markdown",
  "page_selection": "range",
  "page_range": "1-3"
}

Document Reading

The Document Reading endpoint focuses on extracting clean, structured text from documents. This service is optimized for text-heavy documents and provides high-quality text extraction with formatting.

Endpoint

POST /docint/read Extract printed and handwritten text from images and documents

Request Parameters

  • file_ids (required) - Array of file IDs to process (uploaded via /file endpoint)
  • barcode_detection (optional) - Enable barcode detection (default: false)
  • language (optional) - Include language identification (default: false)
  • page_selection (optional) - Page selection strategy (all_pages or range, default: all_pages)
  • page_range (optional) - Page range to process (e.g., "1,3-6"), only used if page_selection is "range"

Supported File Types

  • PDF (.pdf)
  • Images (.jpg, .jpeg, .png, .tiff, .tif)

Example Request

POST /docint/read
Headers:
  X-Token: your_token_here
  Content-Type: application/json

{
  "file_ids": ["f7d23a1e-8c5b-4f9a-b102-3d4e91c8a7b2"],
  "language": true,
  "page_selection": "all_pages"
}

Document Summarization

The Document Summarization endpoint analyzes and summarizes document content. It provides key points, document structure, and concise summaries of lengthy documents.

Endpoint

POST /docint/summarization Summarize documents using AI (PDF, DOCX, PPTX, XLSX)

Request Parameters

  • file_id (required) - ID of the uploaded file to summarize
  • length (optional) - Desired summary length (short, medium, long, very_long, default: medium)
  • style (optional) - Style of the summary (concise, detailed, creative, technical, narrative, bullet_points, default: concise)
  • include_structure (optional) - Whether to include document structure in the summary (default: true)
  • temperature (optional) - Creativity level (0.0 = deterministic, 1.0 = creative, default: 0.3)
  • include_file_upload (optional) - Whether to create and upload a text file with the summary (default: true)

Supported File Types

  • PDF (.pdf)
  • Microsoft Word (.docx)
  • Microsoft PowerPoint (.pptx)
  • Microsoft Excel (.xlsx, .xls)

Example Request

POST /docint/summarization
Headers:
  X-Token: your_token_here
  Content-Type: application/json

{
  "file_id": "a9c8e7d6-5b4a-3c2d-1e0f-9g8h7i6j5k4l",
  "length": "medium",
  "style": "concise",
  "include_structure": true,
  "temperature": 0.3
}

Special Handling for Document Types

The summarization endpoint adapts its processing based on document type:

  • PDF - Extracts text across all pages, preserving document structure
  • DOCX - Extracts text, tables, and embedded content
  • PPTX - Processes slides to capture presentation flow and content
  • XLSX - Extracts data from worksheets, focusing on trends and key metrics

Pro Tip

For optimal summarization:

  • Set length to medium for most business documents
  • Use style: "technical" for scientific or technical papers
  • For presentations, style: "bullet_points" works best
  • Set temperature lower (0.1-0.3) for more factual summaries, higher (0.6-0.8) for more creative ones
