Overview

These services now utilize an asynchronous job processing architecture, making them more efficient and capable of handling larger audio files with improved reliability.

How It Works - Asynchronous Processing

Our speech services use an asynchronous job-based architecture to efficiently process audio files of any size:

Speech-to-Text Workflow

Step 1
Upload your audio file using the /file endpoint
Step 2
Submit a job with the file_id to the speech service endpoint
Step 3
Poll the job status endpoint using the returned job_id
Step 4
Retrieve the transcription results when job is completed

The key advantages of our asynchronous job processing system:

  • Improved reliability: Audio processing continues even if your client disconnects
  • Better scalability: Efficiently handles multiple requests and large audio files
  • No timeouts: Perfect for processing long audio files that would timeout with synchronous APIs
  • Progress tracking: Check status at any time through the job status endpoint

File Format Requirements

  • Supported formats: WAV, MP3, M4A, FLAC, OGG
  • Maximum file size: 200MB
  • Maximum audio length: 2 hours per single audio file
  • Recommended sampling rate: 16kHz or higher
  • Channels: Mono or stereo (diarization works best with clear audio)

Job Management API

Our speech services use an asynchronous job-based architecture. After submitting a job, use these endpoints to check status and retrieve results.

Check Job Status

GET /jobs/status?job_id={job_id}

Check the current status of your job (pending, processing, completed, or failed). Use this to poll for completion.

Required headers:

  • X-Token: Your authentication token

Response includes status, timestamps, and error message (if any).

Retrieve Job Results

GET /jobs/result?job_id={job_id}

Once your job status is "completed", use this endpoint to retrieve the full transcription results.

Required headers:

  • X-Token: Your authentication token

Returns the complete transcription data including timing information, speaker identification (if requested), and any other job-specific outputs.

List Jobs

GET /jobs

View a list of your jobs with pagination and filtering options.

Optional query parameters:

  • limit: Maximum number of jobs to return (default: 10)
  • offset: Offset for pagination (default: 0)
  • status: Filter by job status
  • job_type: Filter by job type (e.g., "stt", "stt_diarize")

Job Status Polling Best Practices

When polling for job completion, implement an exponential backoff strategy to avoid overwhelming the API:

  • Start with a 2-3 second interval for short audio files
  • For longer audio files (>1 minute), start with 5-10 seconds
  • Double the interval after each check, up to a maximum of 60 seconds
  • Include a reasonable timeout for very long processes (e.g., 30 minutes)

Pro Tips

For Best Results with Speaker Diarization

  • Use high-quality audio recordings with minimal background noise
  • Ensure speakers don't talk over each other frequently
  • Try to have distinct voice differences between speakers for better identification
  • When possible, use a separate microphone for each speaker

Client Timeout Considerations

Our asynchronous processing model eliminates client timeout issues that can occur with synchronous APIs. This is particularly important for:

  • Long audio files (>15 minutes) that would timeout with synchronous processing
  • Network environments with unstable connections
  • Mobile applications where background processing is needed

The job continues processing even if your client disconnects, and you can retrieve results at any time.

Complete Integration Example

Here's a complete workflow showing how to use our asynchronous speech services from file upload to results retrieval:

Step 1: Upload Audio File

# cURL request curl -X POST https://api.example.com/file \ -H "X-Token: your_token_here" \ -H "Content-Type: multipart/form-data" \ -F "files=@witness_recording.mp3" # Response { "message": "Files uploaded successfully", "uploaded_files": [ { "file_name": "witness_recording.mp3", "file_id": "a1b2c3d4-5678-90ab-cdef-1234567890ab", "content_type": "audio/mpeg" } ] }

Step 2: Submit Transcription Job

# cURL request curl -X POST https://api.example.com/speech/stt_diarize \ -H "X-Token: your_token_here" \ -H "Content-Type: application/json" \ -d '{"file_id": "a1b2c3d4-5678-90ab-cdef-1234567890ab"}' # Response { "message": "Speech-to-text diarization job submitted successfully", "job_id": "23456789-2345-2345-2345-234567890123" }

Step 3: Poll for Job Completion

# Example pseudocode (JavaScript/Node.js) async function waitForJobCompletion(jobId, token) { let backoffTime = 3000; // Start with 3 seconds const maxBackoff = 60000; // Maximum 1 minute between checks const timeout = 30 * 60 * 1000; // 30 minute timeout const startTime = Date.now(); while (Date.now() - startTime < timeout) { // Check job status const response = await fetch(`https://api.example.com/jobs/status?job_id=${jobId}`, { headers: { 'X-Token': token } }); const jobStatus = await response.json(); if (jobStatus.status === 'completed') { return true; // Job is done } else if (jobStatus.status === 'failed') { throw new Error(`Job failed: ${jobStatus.error_message}`); } // Wait with exponential backoff await new Promise(resolve => setTimeout(resolve, backoffTime)); backoffTime = Math.min(backoffTime * 2, maxBackoff); } throw new Error('Job processing timed out'); }

Step 4: Retrieve Results

# cURL request curl -X GET "https://api.example.com/jobs/result?job_id=23456789-2345-2345-2345-234567890123" \ -H "X-Token: your_token_here" # JavaScript example async function getJobResults(jobId, token) { const response = await fetch(`https://api.example.com/jobs/result?job_id=${jobId}`, { headers: { 'X-Token': token } }); if (response.status === 409) { throw new Error('Job is still processing'); } const data = await response.json(); if (data.status === 'completed') { // Process the results const transcript = data.result.enhanced_transcript || data.result.transcript; return transcript; } else { throw new Error(`Job failed: ${data.error_message}`); } }
