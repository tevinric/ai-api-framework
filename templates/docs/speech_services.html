Speech Services API

Transform audio content into accurate text transcriptions with our powerful speech recognition services. From basic transcription to advanced speaker identification, unlock the potential of your audio data.

Download Postman Collection Swagger API Docs

Overview

Our Speech Services API provides powerful speech-to-text conversion services using Microsoft's advanced speech recognition technology. These services enable you to transcribe audio content accurately and efficiently, with options for enhanced features like speaker diarization.

Authentication Required

All speech services endpoints require authentication using a token. You must include your token in the X-Token header for all requests.

See the Token Services documentation for details on token generation.

Multiple Audio Formats

Process a wide range of audio file formats including WAV, MP3, M4A, FLAC, and OGG for maximum flexibility.

Speaker Identification

Identify different speakers in conversations with advanced diarization capabilities for meetings and interviews.

Language Support

Transcribe content in multiple languages with Microsoft's high-accuracy speech recognition technology.

How It Works

Our speech services work in conjunction with our file management system to process audio files:

  1. First, upload your audio file using the /file endpoint
  2. Receive a file_id for your uploaded audio
  3. Submit this file_id to the appropriate speech service endpoint
  4. Receive the transcription results

The uploaded file is automatically deleted after processing to avoid storage bloat.

File Format Requirements

  • Supported formats: WAV, MP3, M4A, FLAC, OGG
  • Maximum file size: 200MB
  • Maximum audio length: 2 hours per single audio file
  • Recommended sampling rate: 16kHz or higher
  • Channels: Mono or stereo (diarization works best with clear audio)

Convert audio recordings to text using Microsoft's high-accuracy speech recognition technology with support for multiple languages and formatting options.

Features

  • High-accuracy transcription
  • Multiple language support
  • Profanity filtering options
  • Fast processing for shorter audio files
  • Clean, readable transcripts

API Endpoint

POST /speech/stt

Convert audio to text by sending a POST request to this endpoint with your file ID.

Required parameters:

  • file_id: ID of the uploaded audio file
  • X-Token: Your API authentication token (header)

Optional parameters:

  • language: Language code (defaults to auto-detection)
  • profanity_filter: Enable/disable profanity filtering

Enhanced Speech-to-Text with Diarization
3 AI Credits

Advanced transcription with speaker identification and timestamps, perfect for meetings, interviews, and multi-speaker scenarios.

Features

  • Speaker identification (Speaker 1, Speaker 2, etc.)
  • Accurate timestamps for each speaker turn
  • AI-enhanced formatting with natural paragraph breaks
  • Handles long audio recordings with automatic chunking
  • Improved accuracy for conversational context
  • Multiple language support

API Endpoint

POST /speech/stt_diarize

Transcribe multi-speaker audio with diarization by sending a POST request to this endpoint with your file ID.

Required parameters:

  • file_id: ID of the uploaded audio file
  • X-Token: Your API authentication token (header)

Optional parameters:

  • language: Language code (defaults to auto-detection)
  • min_speakers: Minimum number of speakers to detect
  • max_speakers: Maximum number of speakers to detect
  • profanity_filter: Enable/disable profanity filtering

Pro Tips

For Best Results with Speaker Diarization

  • Use high-quality audio recordings with minimal background noise
  • Ensure speakers don't talk over each other frequently
  • For longer recordings (>30 minutes), consider splitting the file into smaller segments
  • When possible, use a separate microphone for each speaker
  • Provide min_speakers and max_speakers parameters if you know the approximate number of participants
