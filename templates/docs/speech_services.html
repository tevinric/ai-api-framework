Speech Services

Our API platform provides powerful speech-to-text conversion services using Microsoft's advanced speech recognition technology. These services enable you to transcribe audio content accurately and efficiently, with options for enhanced features like speaker diarization.

Download Postman Collection

Authentication Required

All speech services endpoints require authentication using a token. You must include your token in the X-Token header for all requests.

See the Token Services documentation for details on token generation.

Available Services

Basic Speech-to-Text

1 credit

Convert audio recordings to text using Microsoft's high-accuracy speech recognition technology.

  • High-accuracy transcription
  • Multiple language support
  • Profanity filtering options
POST /speech/stt

Enhanced Speech-to-Text with Diarization

3 credits

Advanced transcription with speaker identification and timestamps, perfect for meetings and interviews.

  • Speaker identification (Speaker 1, Speaker 2, etc.)
  • Accurate timestamps for each speaker turn
  • AI-enhanced formatting with natural paragraph breaks
  • Handles long audio recordings with automatic chunking
POST /speech/stt_diarize

How It Works

Our speech services work in conjunction with our file management system to process audio files:

  1. First, upload your audio file using the /file endpoint
  2. Receive a file_id for your uploaded audio
  3. Submit this file_id to the appropriate speech service endpoint
  4. Receive the transcription results

The uploaded file is automatically deleted after processing to avoid storage bloat.

File Format Requirements

  • Supported formats: WAV, MP3, M4A, FLAC, OGG
  • Maximum file size: 100MB
  • Recommended sampling rate: 16kHz or higher
  • Channels: Mono or stereo (diarization works best with clear audio)

Example Usage

Step 1: Upload Audio File

POST /file Header: X-Token: your_token_here Content-Type: multipart/form-data Form data: - files: [your_audio_file.mp3] Response: { "message": "Files uploaded successfully", "uploaded_files": [ { "file_name": "meeting_recording.mp3", "file_id": "a1b2c3d4-5678-90ab-cdef-1234567890ab", "content_type": "audio/mpeg" } ] }

Step 2: Submit for Transcription with Diarization

POST /speech/stt_diarize Header: X-Token: your_token_here Content-Type: application/json { "file_id": "a1b2c3d4-5678-90ab-cdef-1234567890ab" } Response: { "message": "Audio processed successfully", "raw_transcript": "Hello everyone welcome to our weekly meeting. I'd like to discuss our progress on the Henderson project. Sure, I can provide an update on that. We've completed the initial analysis and are now moving into the implementation phase.", "enhanced_transcript": "[00:00:00] Speaker 1: Hello everyone welcome to our weekly meeting. I'd like to discuss our progress on the Henderson project.\n\n[00:00:07] Speaker 2: Sure, I can provide an update on that. We've completed the initial analysis and are now moving into the implementation phase." }

Pro Tip

For best results with speaker diarization:

  • Use high-quality audio recordings with minimal background noise
  • Ensure speakers don't talk over each other frequently
  • For longer recordings (>30 minutes), consider splitting the file into smaller segments
