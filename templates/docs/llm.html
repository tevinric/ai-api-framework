Complete LLM Integration Guide

Learn how to integrate multimodal capabilities, context files, and advanced features into your applications

Working with Images in LLMs

GPT-4o and GPT-4o-mini support multimodal inputs, allowing you to send images along with text prompts for comprehensive analysis.

Step 1: Upload Images

First, upload your images using the File Management API. Only PNG, JPG, and JPEG formats are supported.

# Upload an image file curl -X POST "https://api.gaia.com/files/upload" \ -H "X-Token: your_token_here" \ -F "file=@image.png" \ -F "description=Product image for analysis"

The response will include a file_id that you'll use in your LLM request:

{ "file_id": "12345678-1234-1234-1234-123456789012", "file_name": "image.png", "content_type": "image/png", "file_size": 2048576 }

Step 2: Send Multimodal Request

Include the file_ids array in your LLM request to enable image analysis alongside your text prompt.

curl -X POST "https://api.gaia.com/llm/gpt-4o" \ -H "X-Token: your_token_here" \ -H "Content-Type: application/json" \ -d '{ "system_prompt": "You are an expert image analyst.", "user_input": "Analyze this product image and describe its features.", "file_ids": ["12345678-1234-1234-1234-123456789012"], "temperature": 0.7 }'

Image Limits

GPT-4o: Maximum 20 images per request
GPT-4o-mini: Maximum 10 images per request
Supported formats: PNG, JPG, JPEG only

Token Estimation

Each image consumes approximately 765 tokens from your context window. The total request size (text + images) is checked against the model's context limit before processing.

Using Context Files for Enhanced Responses

Context files allow you to provide additional background information that enhances the LLM's responses without including it directly in your prompt.

Step 1: Create a Context File

Upload content or files to create a searchable context that can be referenced in LLM calls.

curl -X POST "https://api.gaia.com/context" \ -H "X-Token: your_token_here" \ -H "Content-Type: application/json" \ -d '{ "content": "Company ABC provides AI solutions for businesses...", "name": "Company Knowledge Base", "description": "Internal knowledge about company products and services" }'

Step 2: Reference Context in LLM Calls

Use the context_id parameter (supported by GPT-4o) to automatically enhance your system prompt with relevant context.

curl -X POST "https://api.gaia.com/llm/gpt-4o" \ -H "X-Token: your_token_here" \ -H "Content-Type: application/json" \ -d '{ "system_prompt": "You are a helpful assistant.", "user_input": "Tell me about our AI solutions.", "context_id": "context-uuid-here", "temperature": 0.5 }'

How Context Integration Works

When you provide a context_id, GAIA automatically retrieves the context content and appends it to your system prompt with clear separators. This allows the LLM to reference the additional information when generating responses.

Step 3: Manage Context Files

You can update, retrieve, and delete context files as needed:

# List all contexts curl -X GET "https://api.gaia.com/context/list" \ -H "X-Token: your_token_here" # Update context content curl -X PUT "https://api.gaia.com/context" \ -H "X-Token: your_token_here" \ -H "Content-Type: application/json" \ -d '{ "context_id": "your-context-id", "content": "Updated content...", "append": true }' # Delete a context curl -X DELETE "https://api.gaia.com/context?context_id=your-context-id" \ -H "X-Token: your_token_here"

LLM Best Practices and Optimization

Choosing the Right Model

  • GPT-4o: Complex reasoning, multimodal analysis, context-heavy tasks
  • GPT-4o-mini: Quick responses, cost-effective applications, simple image analysis
  • O1-mini: Mathematical problems, code debugging, logical reasoning
  • O3-mini: Long-form content, configurable reasoning depth
  • DeepSeek-R1: Technical analysis, research tasks, chain-of-thought reasoning
  • DeepSeek-V3: General-purpose applications, API integrations
  • Llama 3.1: Open-ended conversations, creative content, multilingual tasks

Temperature and Parameter Optimization

Adjust model parameters based on your use case:

  • Low temperature (0.1-0.3): Factual responses, consistent outputs
  • Medium temperature (0.4-0.7): Balanced creativity and consistency
  • High temperature (0.8-1.0): Creative writing, diverse outputs

Context Window Management

Monitor your token usage to avoid context window limits:

  • Each image ≈ 765 tokens
  • Text ≈ 1 token per 4 characters
  • Reserve tokens for model response
  • Consider pagination for large inputs

Cost Optimization Strategies

  • Use GPT-4o-mini for development and testing
  • Implement request caching for repeated queries
  • Optimize prompt length while maintaining clarity
  • Use context files instead of repeating information in prompts
  • Choose appropriate models based on task complexity
