Language Models (LLMs)

Our API platform provides access to several state-of-the-art language models for a wide range of text generation and understanding tasks. Each model has different capabilities and cost structures to suit your specific needs.

Download Postman Collection

Authentication Required

All LLM endpoints require authentication using a token. You must include your token in the X-Token header for all requests.

See the Token Services documentation for details on token generation.

Available Models

GPT-4o

2 credits

OpenAI's most advanced model, offering superior text comprehension and generation for complex tasks.

  • Excellent for complex reasoning
  • Supports JSON output format
  • Handles sophisticated instructions
POST /llm/gpt-4o

GPT-4o-mini

0.5 credits

A lightweight version of GPT-4o, suitable for everyday tasks at a lower cost.

  • Most cost-effective option
  • Good for general content generation
  • Ideal for development and testing
POST /llm/gpt-4o-mini

Deepseek-R1

3 credits

Specialized for deep reasoning and complex problem-solving.

  • Excels at multi-step reasoning
  • Performs well on technical content
  • Capable of detailed analysis
POST /llm/deepseek-r1

Llama

3 credits

Meta's Llama model with 405B parameters for high-quality text generation.

  • Excellent context understanding
  • Strong creative writing capabilities
  • Handles diverse topics well
POST /llm/llama

O1-mini

5 credits

Latest-generation model for advanced reasoning and complex tasks.

  • Superior chain-of-thought reasoning
  • Excellent for complex problem-solving
  • High accuracy on technical content
POST /llm/o1-mini

Model Selection Guide

When choosing which model to use, consider the following factors:

  • Complexity of task - For more complex reasoning or specialized tasks, use GPT-4o, Deepseek-R1, or O1-mini
  • Budget constraints - For cost-sensitive applications, start with GPT-4o-mini
  • Response quality vs. speed - Balance the quality of responses against response time based on your application needs

For complete details and examples, please refer to our Swagger Documentation.
