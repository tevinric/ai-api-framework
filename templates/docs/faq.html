Authentication Questions

How do I authenticate with the API?

We use a two-step authentication process:

  1. First, you authenticate with your unique API key to generate a token by calling the /token endpoint
  2. Then, you use this token in the X-Token header for all subsequent API calls

This approach enhances security by limiting the exposure of your API key and providing temporary access tokens.

How long are authentication tokens valid?

Authentication tokens are valid for 1 hour (3599 seconds) from the time of generation. After this period, you'll need to generate a new token or refresh the existing one to continue using the API.

How do I refresh my authentication token?

You can refresh your token by calling the /token/refresh endpoint with your API key in the header and the current token as a query parameter. This will extend the token's validity for another hour without requiring a completely new token generation.

What should I do if my API key is compromised?

If you believe your API key has been compromised, contact the AI COE team immediately to revoke the existing key and issue a new one. For security reasons, never share your API key or include it in client-side code or public repositories.

Services Questions

Which language models are available on the platform?

We offer several cutting-edge language models:

  • GPT-4o: OpenAI's latest model for complex content generation and reasoning
  • GPT-4o-mini: Lightweight version for everyday tasks
  • O1-mini: Specialized for complex reasoning and problem-solving
  • O3-mini: Text generation with variable reasoning settings
  • DeepSeek-R1: Focused on deep reasoning capabilities
  • DeepSeek-V3: General-purpose text generation
  • Llama 3.1 (405B): Meta's large language model

Each model has different strengths, costs, and use cases. Visit our Language Models page for detailed information.

What's the difference between the LLM and Conversational endpoints?

The LLM endpoints are designed for one-off text generation, while the Conversational endpoints maintain context across multiple messages:

  • LLM endpoints (/llm/*): Best for single queries, text generation, and tasks that don't require maintaining conversation history
  • Conversational endpoints (/conversation/*): Optimized for multi-turn conversations where context from previous interactions is important

Use Conversational endpoints when building chatbots or applications that need to remember previous user interactions.

Which image generation models are available?

We offer two advanced image generation models:

  • DALL-E 3: OpenAI's powerful image generation model, excellent for highly detailed and accurate representations of text prompts
  • Stable Diffusion Ultra: Advanced open-source model that specializes in high-quality, creative image generation

Both models allow you to create images from text descriptions, but they have different strengths and aesthetic styles. See our Image Generation page for details and examples.

What's RAG and how can I use it?

RAG (Retrieval-Augmented Generation) allows you to build AI applications that can answer questions based on your specific content. It combines vector search with language models to provide accurate, context-aware responses grounded in your documents.

To use RAG:

  1. Upload your documents to our platform
  2. Create a RAG collection using those documents
  3. Query the collection with natural language questions

RAG is ideal for building knowledge bases, question-answering systems, and customer support tools based on your organization's specific information. See our RAG Services page for implementation details.

Pricing & Credits Questions

How does the credit system work?

Our platform uses a credit-based system:

  • Each API endpoint consumes a specific number of AI credits per call
  • Credits are allocated monthly based on your account tier
  • Different models and services have different credit costs based on their computational requirements

For example, GPT-4o costs 2 credits per call, while GPT-4o-mini costs only 0.5 credits. The credit system helps manage resource allocation and ensures fair access to AI services.

What account tiers are available?

We offer several account tiers with varying credit allocations:

  • Test/Trial: 100 monthly AI credits for initial exploration
  • Developer: 999 monthly AI credits for development and testing
  • Production Tier 3 - Basic: 5,000 monthly AI credits for low-volume production applications
  • Production Tier 2 - Professional: 10,000 monthly AI credits for medium-volume production applications
  • Production Tier 1 - Professional: 25,000 monthly AI credits for high-volume production applications

You can specify your requirements when requesting an API key, and our team will set up your account accordingly.

How can I check my current credit balance?

You can check your current AI credit balance using the /usage/balance endpoint. This will return information about your current balance, total allocated credits, and usage statistics for the current month.

We also provide detailed usage statistics through our Balance Management endpoints, which allow you to track consumption across different services over time.

What happens if I run out of credits?

If you run out of credits, your API calls will return an error message indicating insufficient credits. Credits reset at the beginning of each month according to your account tier.

If you need additional credits before the reset, contact the AI COE team to discuss your requirements. We may be able to allocate additional credits based on your use case and business needs.

File Management Questions

What file types are supported?

Our platform supports a wide range of file formats across different categories:

  • Documents: PDF, DOCX, TXT, RTF, CSV, JSON
  • Images: JPEG, PNG, GIF, BMP, SVG
  • Audio: MP3, WAV, M4A, FLAC, OGG
  • Other: ZIP, XML, JSON, CSV

The specific formats supported may vary depending on the service you're using. Check the documentation for each service for specific format requirements.

Are there any file size limits?

Yes, we have the following file size limitations:

  • Maximum file size: 200MB per file
  • For speech-to-text services: Maximum audio length of 2 hours
  • For document intelligence: Maximum of 100 pages per document

For larger files, consider splitting them into smaller chunks or contact our support team for assistance with processing large volumes of data.

How long are uploaded files stored?

Files are stored differently depending on the service:

  • Files used with certain services (like speech-to-text) are automatically deleted after processing
  • Other files remain available until explicitly deleted by the user or until they expire (typically 30 days)

You can manually delete files at any time using the /file endpoint with the DELETE method. We recommend removing files once they're no longer needed.

How do I manage my uploaded files?

You can manage your files using these endpoints:

  • POST /file: Upload one or more files
  • GET /file/url: Get access URL for a previously uploaded file
  • DELETE /file: Delete a previously uploaded file
  • GET /file/list: List all files uploaded by your account

We also provide a visual File Portal interface for easy file management. Access it via the File Portal link in the sidebar navigation.

Technical Questions

Are there rate limits for API calls?

Yes, we implement rate limits to ensure fair usage and system stability:

  • Default rate limit: 60 requests per minute per user
  • Burst rate limit: 10 requests per second

Rate limits may vary by service and account tier. If you hit a rate limit, your request will receive a 429 (Too Many Requests) response. Implement appropriate retry logic with exponential backoff in your applications.

How should I handle errors from the API?

Our API uses standard HTTP response codes to indicate success or failure. Here are common error codes and how to handle them:

  • 401 Unauthorized: Authentication failed (invalid or expired token). Generate a new token.
  • 403 Forbidden: Insufficient permissions. Check your account scope.
  • 400 Bad Request: Invalid parameters or request format. Check your request body.
  • 429 Too Many Requests: Rate limit exceeded. Implement backoff and retry.
  • 500 Server Error: Unexpected server error. Retry after a delay or contact support.

All API errors include a message field explaining the issue. Implement proper error handling and logging in your applications.

How can I optimize my API usage?

Here are some best practices for optimizing your API usage:

  1. Reuse authentication tokens within their validity period (1 hour) instead of generating new ones for each request
  2. Start with cost-effective models (like GPT-4o-mini) for development and testing
  3. Use the appropriate model for each task (don't use an advanced model when a simpler one would suffice)
  4. Implement caching for responses that don't change frequently
  5. Monitor your usage with the Balance Management endpoints to identify optimization opportunities
  6. For batch processing, consider using our asynchronous endpoints where available

These practices can help reduce costs and improve application performance.

Do you provide client libraries or SDKs?

Currently, we don't provide official client libraries or SDKs. However, our API is RESTful and follows standard conventions, making it easy to integrate with any programming language that supports HTTP requests.

We provide a comprehensive Postman Collection that you can download and import into Postman to quickly get started. The collection includes all endpoints with example requests and response schemas.

Our API documentation includes code examples in popular languages like Python, JavaScript, and cURL to help you integrate with our services.
