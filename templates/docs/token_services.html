Token Services

The Token Services endpoints allow you to generate, refresh, and manage authentication tokens for API access. These tokens are required for all other API calls.

Authentication Flow

Our API uses a two-step authentication process:

  1. First, you authenticate with your API key to generate a token
  2. Then, you use this token for subsequent API calls

Tokens expire after a period (typically one hour) and can be refreshed before expiration.

Available Endpoints

POST /token Generate a new authentication token
GET /token/details Get details about a token
POST /token/refresh Refresh an existing token

For complete details and examples, please refer to our Swagger Documentation or download our Postman Collection.
