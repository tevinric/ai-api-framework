Token Services

GAIA's Token Service endpoints allow you to generate, refresh, and manage authentication tokens for API access. Valid tokens are required for all AI API calls. Follow this guide to understand our token-based authentication system.

Authentication Flow

Understanding the two-step authentication process

1

Generate a Token with Your API Key

Use your unique API key to generate an authentication token by calling the /token endpoint. Your API key is passed in the request header.

Token Generation GET /token
// Include API key in header
Headers: {
"API-Key": "your_api_key_here"
}
// Example response
{
"access_token": "eyJhbGciOiJIUzI1NiIsInR5...",
"token_type": "Bearer",
"expires_in": 3599,
"expires_on": "2025-03-31 15:30:45 +0200"
}
2

Use Token for API Requests

Include the generated token in the X-Token header for all subsequent API calls. This token authenticates your requests to our AI services.

API Request with Token POST /llm/gpt-4o-mini
// Include token in header
Headers: {
"X-Token": "eyJhbGciOiJIUzI1NiIsInR5...",
"Content-Type": "application/json"
}

// Request body with your data
Body: {
"user_input": "Tell me about AI APIs"
}
3

Refresh Token Before Expiration (Optional)

Tokens expire after 1 hour (3599 seconds). You can refresh a token before it expires to extend its validity period without generating a new one.

Token Refresh GET /token/refresh
// Include API key and token
Headers: {
"API-Key": "your_api_key_here"
}

Query Parameters: {
"token": "eyJhbGciOiJIUzI1NiIsInR5..."
}

Authentication Flow

GAIA uses a two-step authentication process:


  1. First, you authenticate with your unique API key to generate a token
  2. Then, you use this token for subsequent API calls

Tokens expire after 1 hour (3599 seconds) and can be refreshed before and after expiration.

You can use the same token to call multiple AI endpoints in your application flow as long as your process remains within the token validity period.

Available Endpoints

Detailed information about each Token Service endpoint

Endpoint Description
GET
/token

Generate Token

Generate a new authentication token using your API key. This token will be valid for 1 hour and is required for all API calls.

Request Headers

API-Key Your unique API key (required)

Response

access_token The generated token to use for API calls
token_type Type of token (always "Bearer")
expires_in Time in seconds until token expiration (3599)
expires_on Timestamp of token expiration
GET
/token/details

Get Token Details

Retrieve details about a specific token, including its status (valid, invalid, or expired), user ID, scope, and expiration time.

Request Headers

API-Key Your unique API key (required)

Query Parameters

token The token to get details for (required)

Response

token_status Current status: "valid", "invalid", or "expired"
user_id ID of the user who issued the token
token_scope Scope of the token
token_expiration_time Token expiration timestamp
GET
/token/refresh

Refresh Token

Refresh an existing token to extend its validity period. This generates a new token while keeping the same permissions and user context.

Request Headers

API-Key Your unique API key (required)

Query Parameters

token The token to refresh (required)

Response

access_token The new refreshed token
token_type Type of token (always "Bearer")
expires_in Time in seconds until token expiration (3599)
expires_on Timestamp of token expiration

Best Practices

Recommendations for efficient token management

Token Management Tips

1

Reuse tokens

Use the same token for multiple API calls within the same workflow or process stream (until it expires)

2

Use token refreshing

Instead of generating new tokens, refresh existing ones when possible (only if required)

3

Track expiration

Monitor the expiration time and refresh before tokens expire

4

Keep tokens secure

Store tokens securely and never share them in client-side code

5

Token Generation

Generated new tokens for each new session or process work stream

Token Lifecycle

Understanding the token lifecycle can help you implement authentication more efficiently:

1

Generation

A token is generated using your API key

2

Usage

The token is used for API calls for up to 1 hour

3

Refreshing

Before expiration, the token can be refreshed for a new 1-hour period

4

Expiration

After 1 hour, if not refreshed, the token expires and becomes invalid

Error Handling

Common authentication errors and how to handle them:

401 Unauthorized

Token is missing, invalid, or expired. Generate a new token to resolve.

400 Bad Request

Missing required parameters or invalid format. Check your request body.

403 Forbidden

Token doesn't have sufficient permissions. Check user scope.

500 Server Error

Unexpected server error. Retry the request or contact support.
